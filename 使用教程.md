# girlfriend-agent 使用教程

## 一、启动服务

```bash
cd A:/claudeworks/女友agent
pip install -r requirements.txt
python -m src.engine_server
```

服务默认在 http://127.0.0.1:18012 启动，首次启动会自动初始化 `~/.girlfriend-agent/` 数据目录并创建 git 仓库。

## 二、核心理念

女友 agent 不是直接回复用户的人——它是一个**人格状态机**。调用方（大模型 agent）拿到引擎返回的信息（prompt + 记忆 + 去AI味指令 + 图谱关联），然后把这些注入到模型中，模型据此生成回复。

**完整使用流程**：

1. 选人设 → `POST /persona/apply-template`
2. 每次对话 → `POST /chat` → 得到 prompt、记忆、去AI味指令
3. 把返回的 prompt 注入大模型 → 大模型生成人格化回复
4. 从大模型回复中提取关键事实 → `POST /memory/update` 写入记忆
5. 从大模型对话中提取实体和关系 → `POST /graph/add-entity` / `POST /graph/add-relation` 构建情景图谱
6. 每7次对话 → `POST /evolve` 执行进化微调
7. 查看进化方向 → `GET /evolve/direction` 看你正在走向哪个结局

## 六、Level 注入等级

level 参数控制每次对话注入多少上下文给模型：

| Level | 记忆数 | 附加内容 | 字数 | 何时使用 |
|-------|--------|----------|------|----------|
| 1 | 3条 | 人格摘要 | ~600字 | 普通对话 |
| 2 | 8条 | 关系状态+情绪趋势+图谱关联 | ~2500字 | 需要更深入的回复 |
| 3 | 15条 | 完整进化状态+图谱遍历+原始对话 | ~5000字 | 需要深度对话 |

## 七、情景图谱使用

情景图谱（LazyGraphRAG）用于存储实体间的关联关系，支持因果推理和时间线追踪。

**典型用法**：

```bash
# 从对话中提取实体
POST /graph/add-entity {"entity_name": "实验进度", "entity_type": "topic"}
POST /graph/add-entity {"entity_name": "压力大", "entity_type": "emotion"}

# 连接实体和事件
POST /graph/add-event {"description": "因为赶实验所以压力大", "entities": ["实验进度", "压力大"], "relation_type": "caused"}

# 搜索图谱关联
POST /graph/search {"query": "实验", "max_depth": 3}
# 返回：实验进度 → caused → 压力大 → related_to → 需要安慰

# 每7次对话后批量建图
POST /graph/batch-build {"session_count": 7}
```

**图谱 vs 向量记忆对比**：

| 问题 | 向量记忆 | 情景图谱 |
|------|----------|----------|
| "他喜欢什么？" | 能回答 | 能但更慢 |
| "上次讨论X时他选了什么？" | 无法追踪时间线 | 图遍历 |
| "X和Y之间有什么关系？" | 无法推理关联 | 关系推理 |
| "为什么他会这样？" | 无法推理因果 | 因果链 |

## 八、进化系统详解

### 进化周期

每7次对话触发一次进化周期，流程如下：

1. **观察模式** — 分析近7次对话：话题分布、情绪基调、隐性需求
2. **情境驱动微调** — 根据观察结果调整人格参数：
   - 用户压力大 → warmth+0.1, proactivity+0.05
   - 用户开心 → humor+0.1
   - 需要深度交流 → curiosity+0.08
3. **安全约束** — 单次调整≤10%，连续3次同方向→减半，5次→停止
4. **记录** — 写入 evolution_log + git commit

### 56种进化结局

8个属性×7个副属性 = 56种独特结局。例如：

| 方向 | 特征属性 | 最终形态 |
|------|----------|----------|
| 温暖守护型 | 关心+细腻 | 最懂你心事的人 |
| 活力搭档型 | 表达+幽默 | 让你每天都想聊的人 |
| 默契知己型 | 理解+记忆 | 最默契的伙伴+恋人 |
| 勇敢战士型 | 勇气+关心 | 会为你冲锋也会温柔照顾 |

### 回退机制

- `POST /evolve/revert` — 回退最近一次进化（恢复人格参数）
- `POST /evolve/revert-to` — 回退到指定版本（需要 commit_hash）
- `GET /evolve/history` — 查看所有进化commit历史
- 回退只影响人格参数和进化状态，**记忆数据永久保留**

## 十、完整对话循环示例

```python
import httpx

client = httpx.Client(base_url="http://127.0.0.1:18012")

# 第一次使用：选人设
client.post("/persona/apply-template", json={"template_id": "gentle"})

# 每次对话：
# 1) 获取人格上下文
resp = client.post("/chat", json={
    "user_message": "我实验做到很晚，很累",
    "level": 2,
    "interaction_type": "emotion_companion"
})
context = resp.json()
# context["persona_prompt"] → 注入模型的 prompt
# context["memory_fragments"] → 相关记忆片段
# context["de_ai_instructions"] → 去AI味指令
# context["relationship_summary"] → 关系状态摘要
# context["emotion_trend"] → 情绪趋势
# context["graph_context"] → 图谱关联摘要

# 2) 大模型用 context 生成回复后，提取关键事实写入记忆
client.post("/memory/update", json={
    "content": "用户实验做得晚很累",
    "memory_type": "emotion"
})

# 3) 从对话中提取实体和关系构建图谱
client.post("/graph/add-entity", json={"entity_name": "实验", "entity_type": "topic"})
client.post("/graph/add-entity", json={"entity_name": "疲惫", "entity_type": "emotion"})
client.post("/graph/add-relation", json={
    "source_entity": "实验",
    "target_entity": "疲惫",
    "relation_type": "caused"
})

# 4) 每7次对话执行进化
client.post("/evolve")

# 5) 查看进化方向
direction = client.get("/evolve/direction").json()
# direction["ending"]["name"] → 当前结局名称
# direction["progress"]["overall_progress"] → 进化进度

# 6) 如果进化效果不好，可以回退
client.post("/evolve/revert")
```

## 十二、常见问题

**Q: 引擎需要什么模型？**
A: 引擎本身不调用任何模型。它提供上下文（prompt + 记忆 + 行为指令），由调用方的大模型来生成回复。

**Q: 数据存储在哪里？**
A: `~/.girlfriend-agent/`，首次启动自动创建。chroma_db、session_memory、graphrag_db 三个目录被 git 排除（记忆永久保留），config/ 和 evolution_log/ 被 git 管理（支持回退）。

**Q: 记忆衰减怎么工作？**
A: 基于艾宾浩斯遗忘曲线，公式：`sqrt(访问次数+1) * exp(-0.1 * 天数)`。越久没用的记忆权重越低，越常检索的记忆权重越高。可以通过 `/memory/decay` 手动触发全局衰减。

**Q: 进化微调会失控吗？**
A: 不会。单次调整≤10%，连续3次同方向→幅度减半，连续5次→停止调整。还有回退机制可随时撤销。

**Q: 图谱有什么用？**
A: 向量记忆只能回答"他喜欢什么"这类简单问题。图谱能追踪因果链（"压力大→因为赶实验→实验deadline是4月底"）、推理关联（"火山图和ggplot2之间有什么关系"）、追踪时间线。
